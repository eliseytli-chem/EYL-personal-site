# AGENTS.md

## 專案定位

這是一個以中文為主的個人作家網站，用來長期典藏散文、育兒札記、家庭閱讀、教育思辨，以及未來可能投稿的文章。網站應像安靜、耐讀的個人文庫，而不是追逐流量與即時互動的部落格。

## 技術原則

- 使用 Astro 建立靜態網站，優先採用 Astro 原生能力。
- 文章一律以 Markdown 管理並放在 `src/content/essays/`，使用 Astro Content Collections 定義與驗證內容結構。
- 不引入複雜 CMS、資料庫或不必要的後端服務。
- 優先選擇簡單、穩定、容易理解與長期維護的方案。
- 新增套件前，先確認 Astro、CSS 或少量原生 JavaScript 是否已能完成需求。
- 避免過度工程化；不要為尚未出現的需求預先建立複雜抽象、狀態管理或多層架構。

## 語言與內容

- 網站介面、導覽、說明文字與預設 metadata 皆以繁體中文為主。
- 中文排版與長文閱讀體驗優先；程式碼識別字、檔名及必要技術詞彙可使用英文。
- 保留文章原文與 Markdown/MDX 原始檔的可攜性，避免內容被綁定在專有格式中。
- 文章檔名使用簡潔、穩定的英文 slug（例如 `reading-together.md`），並作為 `/essays/[slug]/` 網址；發布後不要任意變更，必要時應保留重新導向。
- 中文分類名稱應映射至固定英文 slug，避免直接將中文名稱當成 URL path。

## 文章 metadata

所有文章均須支援以下 frontmatter 欄位：

```yaml
title: 文章標題
date: 2026-06-19
category: 散文
tags:
  - 家庭
description: 簡短摘要
visibility: public
featured: false
```

欄位規則：

- `title`：必填，文章標題。
- `date`：必填，使用可被明確解析的日期格式，建議為 `YYYY-MM-DD`。
- `category`：必填，文章主要分類。
- `tags`：必填，可為空陣列；用於補充主題標記。
- `description`：必填，簡潔描述文章內容，供列表、分享與搜尋摘要使用。
- `visibility`：必填，僅接受 `public`、`unlisted`、`private` 或 `draft`。
- `featured`：必填，布林值，表示是否精選。

若使用 Content Collections，應以 schema 驗證以上欄位。除非現有內容另有明確需求，不要任意增加必填 metadata。

## 發布與隱私

- 只有 `visibility: public` 的文章可以出現在首頁、文章列表、分類頁、標籤頁與精選文章。
- `unlisted` 文章可產生靜態文章網址供直接瀏覽，但不得進入首頁、文章列表、分類頁、標籤頁、精選文章、搜尋索引、RSS、sitemap 或相關文章推薦。
- `private` 與 `draft` 文章不得進入文章列表、分類頁、標籤頁、搜尋索引、RSS、sitemap、靜態路由或相關文章推薦。
- 建置文章路由時，必須在資料來源處明確篩選公開文章，不可只在畫面上隱藏連結。
- `featured: true` 不得覆蓋 visibility；非公開文章即使設為精選也不能發布。
- 預覽非公開文章應採用明確且僅限開發環境的機制，不能意外進入正式產物。

## 設計方向

- 視覺應乾淨、安靜、克制，以文字閱讀為核心。
- 重視合宜的字級、行高、段落間距、行寬與留白，確保中文長文舒適易讀。
- 導覽應簡單清楚，優先呈現文章、分類與典藏脈絡。
- 避免資訊密集側欄、跳動元素、彈窗、無限捲動、浮誇動畫及流量導向元件。
- 不以瀏覽數、熱門排行、廣告、成長誘導或社群互動作為設計中心。
- 動畫若存在，應低調並尊重 `prefers-reduced-motion`。
- 在不犧牲閱讀感受的前提下，維持良好的響應式版面、語意化 HTML、鍵盤操作與色彩對比。

## 實作與維護

- 元件保持小而清楚，但只在確實重複或有獨立責任時抽取。
- 優先使用語意化 HTML 與簡單 CSS；少量互動才加入客戶端 JavaScript。
- 避免不必要的 hydration，能以靜態 HTML 完成的內容就保持靜態。
- 變更內容模型、URL、分類或文章檔案結構時，須考慮既有文章與長期連結的相容性。
- 不因小型需求導入大型框架、設計系統、搜尋服務或分析平台。
- 新功能應有明確的閱讀、典藏或維護價值；若成本高於實際收益，選擇較簡單的做法。

## 完成變更前

- 確認正式建置不包含 `private` 或 `draft` 文章，且 `unlisted` 只產生可直接瀏覽的文章頁。
- 確認新增或修改的文章 metadata 符合 schema。
- 執行現有的格式檢查、測試與 Astro build（若專案已提供相關指令）。
- 檢查主要頁面在桌面與行動版的中文閱讀體驗。
- 不修改與任務無關的文章內容或使用者既有變更。
